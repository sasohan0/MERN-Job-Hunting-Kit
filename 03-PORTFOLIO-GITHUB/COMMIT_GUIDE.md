# Commit Message Guide (Short)

Use clear, consistent commits. Recruiters notice.

## Format
`type: short message`

Examples:
- feat: add jwt auth for login
- fix: handle null user profile edge case
- refactor: simplify cart reducer logic
- docs: update README with screenshots
- chore: update dependencies

## Rules
- One commit = one purpose
- Avoid: “update”, “final”, “changes”
